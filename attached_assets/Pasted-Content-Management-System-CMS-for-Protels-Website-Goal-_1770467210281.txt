Content Management System (CMS) for Protels Website
Goal

Build a Content-Only CMS to manage the website content (NO ticketing/support system). The CMS must allow admins to edit content from a control panel and immediately reflect changes on the live website.

1) Admin Panel URL & Access

The CMS admin panel must be accessible at:
/controlpanal

If a user is not authenticated and tries to access /controlpanal, redirect them to the login page.

Use secure authentication (hashed passwords, session/token-based auth, CSRF protection).

2) Default Admin Account (Full Access)

Create a default admin user with full permissions:

Username: Fezzat

Password: Fezzat246810

Role: Super Admin (full access to everything)

Also:

Optionally enforce password strength rules for future accounts.

3) Core Requirement: Live Updates Across All Tabs / Pages
Critical Requirement

All CMS edits must be reflected on the website immediately.

Every “tab” (module/section) in the CMS must be connected to the website templates/components.

When content is updated in the CMS and saved:

The website should show the updated content instantly (or after a very short cache refresh).

If caching is used, implement cache invalidation so changes are not delayed.

The system must support “Preview” (optional) and “Publish” (required), but once published, it must appear on the website without manual code edits.

4) CMS Scope: Content Only (No Ticketing)

The CMS should include:

Pages management

Resorts/Hotels management (as structured content type)

Media library

Global site settings

SEO options

Tracking code settings (GTM)

Do NOT include:

Ticketing / support tickets

Helpdesk modules

Any customer service inbox system

5) Content Modules
A) Pages Module (Static Pages)

Enable full CRUD:

Create / Edit / Delete pages

Page status: Draft / Published

Each page should have:

Page Title

Slug/URL (e.g., /about-us)

Page Content (Rich Text Editor)

Featured image (optional)

“Last updated by” + “Last updated at”

Preview button before publishing (optional but recommended)

B) Resorts/Hotels Module (Structured Content Type)

Create a content type called: Resorts/Hotels (or “Properties”).

Each property should support:

Property Name

Slug (URL)

Short description

Full description (rich editor)

Location text (or address)

Hero image

Gallery (multiple images)

Facilities list (editable items)

Optional: contact info per property (phone/email)

Status: Draft / Published

This module should drive the property pages dynamically, so adding a new property in CMS automatically creates its page on the website.

6) Media Library (Images / Files)

Create a Media Library module:

Upload images (JPG/PNG/WebP) and optionally PDFs

Search by filename

Copy file URL

Delete with warning if the file is currently used in any page/property content

Recommended: auto-generate thumbnails and optimize images for performance

7) SEO Options (Per Page and Per Property)

Add a dedicated SEO panel for each page and each property with:

Meta Title

Meta Description

Meta Keywords (optional)

Canonical URL (optional)

OG Title

OG Description

OG Image (social sharing image)

Robots meta: index/noindex, follow/nofollow

Optional (nice-to-have): Schema/Structured data field

These SEO values must be injected properly into the page <head>.

8) Site Icon (Favicon / App Icons)

Add a setting to upload/manage:

favicon.ico

PNG icons (16x16, 32x32, 180x180 recommended)

Optional: web manifest icons if supported

After upload, the favicon must be applied site-wide automatically.

9) Google Tag Manager (GTM) Code Injection

In Global Settings, add two separate fields:

GTM Head Code

Inject exactly inside <head> before closing </head> on all pages.

GTM Body Code

Inject immediately after the opening <body> tag on all pages.

Requirements:

These codes must be applied globally across the entire website.

Keep the code intact (do not break the GTM script).

Basic validation/sanitization to avoid storing broken HTML while still allowing GTM scripts.

10) Global Settings Module

Create a “Site Settings” section for:

Company name

Phone

Email

Address

Social links (Facebook/Instagram/LinkedIn/etc.)

Logo upload (optional)

Favicon upload (included above)

GTM Head + GTM Body (included above)

All of these must be reflected on the website instantly.

11) Security & Logging (Minimum Required)

Passwords must be stored hashed (bcrypt/argon2).

Protect admin routes (authentication + authorization).

Add a simple audit log:

Who changed what

When it changed

What module/page was edited
